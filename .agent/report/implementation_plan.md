# PH7: AI Integration — Implementation Plan

## Tổng quan

Phân hệ 7 bao gồm **4 bảng** với 2 loại công việc khác nhau:
- **`user_behaviors`**: Thay đổi kiến trúc lớn (Partitioning + composite PK + cột mới)
- **`product_similarities`, `user_recommendations`, `product_associations`**: Brand new tables

> [!IMPORTANT]
> `user_behaviors` yêu cầu **DROP và RECREATE table** vì PostgreSQL không hỗ trợ thêm partitioning vào bảng đang tồn tại. Dữ liệu cũ cần migrate trước khi drop.

> [!WARNING]
> `UserBehaviorRepository` hiện dùng `JpaRepository<UserBehaviorEntity, Long>` — sẽ phải đổi sang composite PK sau khi refactor entity. Kiểm tra toàn bộ caller trước khi sửa.

## Proposed Changes

### Phase 1 — Migration: Recreate `user_behaviors` với Partitioning

#### [NEW] `V5.15__recreate_user_behaviors_partitioned.sql`

```sql
-- Bước 1: Backup data cũ (nếu cần)
-- CREATE TABLE user_behaviors_backup AS SELECT * FROM user_behaviors;

-- Bước 2: Drop bảng cũ và toàn bộ indexes
DROP INDEX IF EXISTS idx_behavior_user;
DROP INDEX IF EXISTS idx_behavior_product;
DROP INDEX IF EXISTS idx_behavior_category;
DROP INDEX IF EXISTS idx_behavior_context;
DROP TABLE IF EXISTS user_behaviors CASCADE;

-- Bước 3: Tạo lại bảng với Partitioning
CREATE TABLE user_behaviors (
    user_behavior_id BIGINT GENERATED BY DEFAULT AS IDENTITY,
    user_id          BIGINT NULL REFERENCES users(user_id) ON DELETE SET NULL,
    session_id       VARCHAR(100) NOT NULL,
    product_id       BIGINT NULL REFERENCES products(product_id) ON DELETE SET NULL,
    category_id      BIGINT NULL REFERENCES categories(category_id) ON DELETE SET NULL,
    action_type      VARCHAR(50) NOT NULL,
    keyword          VARCHAR(255) NULL,
    context_data     JSONB NULL,
    variant_id       BIGINT NULL,
    duration_seconds INT NULL,
    source           VARCHAR(50) NULL,
    created_at       TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_behavior_id, created_at)
) PARTITION BY RANGE (created_at);

-- Bước 4: Tạo indexes mới
CREATE INDEX idx_behavior_user_time    ON user_behaviors(user_id, created_at DESC);
CREATE INDEX idx_behavior_product_time ON user_behaviors(product_id, created_at DESC);
CREATE INDEX idx_behavior_action_time  ON user_behaviors(action_type, created_at DESC);
CREATE INDEX idx_behavior_context      ON user_behaviors USING GIN (context_data);

-- Bước 5: Tạo partitions
CREATE TABLE user_behaviors_y2026m05 PARTITION OF user_behaviors
    FOR VALUES FROM ('2026-05-01') TO ('2026-06-01');
CREATE TABLE user_behaviors_y2026m06 PARTITION OF user_behaviors
    FOR VALUES FROM ('2026-06-01') TO ('2026-07-01');
CREATE TABLE user_behaviors_y2026m07 PARTITION OF user_behaviors
    FOR VALUES FROM ('2026-07-01') TO ('2026-08-01');
```

### Phase 2 — Migration: 3 bảng AI mới

#### [NEW] `V5.16__create_ai_tables.sql`

```sql
-- product_similarities: Kết quả tính độ tương đồng sản phẩm
CREATE TABLE IF NOT EXISTS product_similarities (
    product_id         BIGINT NOT NULL REFERENCES products(product_id) ON DELETE CASCADE,
    similar_product_id BIGINT NOT NULL REFERENCES products(product_id) ON DELETE CASCADE,
    score              DOUBLE PRECISION NOT NULL,
    similarity_type    VARCHAR(50) DEFAULT 'CONTENT_BASED' NOT NULL,
    PRIMARY KEY (product_id, similar_product_id, similarity_type)
);

-- user_recommendations: Cache kết quả gợi ý sản phẩm cho từng user
CREATE TABLE IF NOT EXISTS user_recommendations (
    user_id              BIGINT NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
    product_id           BIGINT NOT NULL REFERENCES products(product_id) ON DELETE CASCADE,
    recommendation_score DOUBLE PRECISION NOT NULL,
    algorithm            VARCHAR(50) DEFAULT 'HYBRID' NOT NULL,
    created_at           TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, product_id, algorithm)
);

-- product_associations: Kết quả Association Rule Mining (MBA)
CREATE TABLE IF NOT EXISTS product_associations (
    product_id         BIGINT NOT NULL REFERENCES products(product_id) ON DELETE CASCADE,
    related_product_id BIGINT NOT NULL REFERENCES products(product_id) ON DELETE CASCADE,
    confidence         DOUBLE PRECISION NOT NULL,
    lift               DOUBLE PRECISION NOT NULL,
    PRIMARY KEY (product_id, related_product_id)
);
```
