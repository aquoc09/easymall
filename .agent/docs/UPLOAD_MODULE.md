# Upload Module — Shared Image Upload System

> **Phạm vi:** Tài liệu mô tả thiết kế, luồng xử lý và cách tích hợp của module Upload vào các module nghiệp vụ trong EasyMall backend.

---

## 1. Tổng quan kiến trúc

Module Upload được xây dựng theo pattern **Decoupled Upload Flow**: FE upload ảnh trước để nhận URL, sau đó gửi URL đó kèm data nghiệp vụ trong một request riêng.

```
┌─────────────────────────────────────────────────────────────┐
│                    Upload Module (Shared)                    │
│                                                             │
│  UploadController  →  StorageService  →  S3StorageServiceImpl│
│       /uploads          (interface)         (AWS S3)        │
│                              ↓                              │
│                    TempUploadCleanupService                  │
│                    (Scheduled @ 2:00 AM)                    │
└─────────────────────────────────────────────────────────────┘
```

### Lý do tách module Upload

| Cách cũ (coupled) | Cách dùng (decoupled) |
|---|---|
| Upload ảnh + tạo entity trong 1 request | Upload ảnh riêng → nhận URL → gửi URL kèm data |
| FE phải gửi form-data lẫn JSON | FE tách 2 bước rõ ràng |
| Khó rollback khi tạo entity thất bại | Ảnh mồ côi được cleanup tự động sau 24h |

## 3. Database — Bảng bổ sung

### 3.1 Bảng `temp_uploads` *(migration V4.9)*

Bảng trung chuyển lưu URL ảnh vừa upload lên S3 nhưng **chưa được liên kết** với entity nghiệp vụ nào.

```sql
CREATE TABLE IF NOT EXISTS temp_uploads (
    id         BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    url        TEXT   NOT NULL UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_temp_uploads_created_at ON temp_uploads (created_at);
```

| Column | Type | Mô tả |
|--------|------|-------|
| `id` | BIGINT (auto) | Primary key |
| `url` | TEXT UNIQUE | URL công khai của ảnh trên S3 |
| `created_at` | TIMESTAMP | Thời điểm upload — dùng để cleanup |

**Vòng đời một bản ghi:**

```
[POST /uploads/image/**]  →  INSERT temp_uploads
        ↓
Nếu FE gửi URL vào entity nghiệp vụ trong 24h  →  URL được dùng, record ở lại
        ↓
Nếu không dùng sau 24h  →  Cleanup job xóa S3 object + xóa record
```

### 3.2 Cột `avatar` trong bảng `users` *(migration V5.0)*

```sql
ALTER TABLE users
    ADD COLUMN IF NOT EXISTS avatar VARCHAR(500) NULL;
```

| Column | Type | Mô tả |
|--------|------|-------|
| `avatar` | VARCHAR(500) NULL | URL đầy đủ của avatar người dùng trên S3 |

## 4. Cấu hình AWS

Tất cả cấu hình lấy từ `application.yaml`, **không hardcode** trong Java:

```yaml
aws:
  base-url: https://shopco-s3.s3.ap-southeast-1.amazonaws.com   # Build URL response
  s3:
    bucket-name: ${AWS_BUCKET}
    region:      ${AWS_DEFAULT_REGION}
  credentials:
    access-key:  ${AWS_ACCESS_KEY_ID}
    secret-key:  ${AWS_SECRET_ACCESS_KEY}
```

> ⚠️ **Yêu cầu S3 Bucket:** Phải tắt **Block Public Access** và bật **ACL** để `PUBLIC_READ` hoạt động. File upload lên sẽ accessible qua URL công khai.

### Hằng số trong `UploadConstants`

| Hằng số | Giá trị | Mô tả |
|---------|---------|-------|
| `MAX_FILE_SIZE_BYTES` | 2 MB | Giới hạn kích thước |
| `ALLOWED_CONTENT_TYPES` | jpeg, png, webp, gif | MIME types được phép |
| `CLEANUP_THRESHOLD_HOURS` | 24 giờ | Ngưỡng xóa ảnh mồ côi |
| `CLEANUP_CRON` | `0 0 2 * * *` | Chạy lúc 2:00 AM hàng ngày |
| `FOLDER_PRODUCTS` | `"products"` | Prefix S3 key cho ảnh sản phẩm |
| `FOLDER_REVIEWS` | `"reviews"` | Prefix S3 key cho ảnh review |
| `FOLDER_USERS` | `"users"` | Prefix S3 key cho avatar user |

## 5. Các Endpoints

> Tất cả endpoints yêu cầu **JWT Bearer Token**.

| Method | Path | Folder S3 | Module dùng |
|--------|------|-----------|-------------|
| POST | `/api/v1/uploads/image/products` | `products/` | Product, ProductVariant |
| POST | `/api/v1/uploads/image/reviews` | `reviews/` | Review |
| POST | `/api/v1/uploads/image/users` | `users/` | User avatar |

**Request (tất cả endpoints đều giống nhau):**
```
POST /api/v1/uploads/image/products
Content-Type: multipart/form-data
Authorization: Bearer <access_token>

Body (form-data):
  key:  file
  type: File
  value: <chọn file ảnh>
```

**Response thành công:**
```json
{
  "code": 200,
  "message": "Image uploaded successfully",
  "result": {
    "url": "https://shopco-s3.s3.ap-southeast-1.amazonaws.com/products/ed5b3da2-dd09-451c-bf78-eae28deb9e27.png"
  }
}
```

**Các lỗi validation:**

| HTTP | Error Code | Nguyên nhân |
|------|------------|-------------|
| 400 | `EMPTY_FILE` | File rỗng hoặc không có file |
| 400 | `INVALID_FILE_TYPE` | MIME type không thuộc danh sách cho phép |
| 400 | `FILE_TOO_LARGE` | File vượt quá 2 MB |
| 500 | `S3_UPLOAD_FAILURE` | Lỗi kết nối AWS hoặc PutObject thất bại |

## 6. Luồng xử lý chi tiết

### 6.1 Flow 2 bước giữa FE và BE

```
╔══════════════════════════════════════════════════════════════╗
║ BƯỚC 1 — Upload ảnh lên S3                                  ║
╠══════════════════════════════════════════════════════════════╣

FE                       Backend                      AWS S3
 │                          │                            │
 │─── POST /uploads/image/**─►                           │
 │    multipart/form-data   │                            │
 │                          ├─ validateFile()            │
 │                          ├─ buildS3Key()              │
 │                          │  → "products/{UUID}.png"  │
 │                          ├─ PutObject (PUBLIC_READ) ──►
 │                          │◄─ 200 OK ─────────────────│
 │                          ├─ INSERT temp_uploads       │
 │◄── { url: "https://..." }─┤                           │


╔══════════════════════════════════════════════════════════════╗
║ BƯỚC 2 — Gửi URL kèm data nghiệp vụ                        ║
╠══════════════════════════════════════════════════════════════╣

FE                       Backend                    Database
 │                          │                           │
 │─── POST /api/v1/products─►                           │
 │    { variantImage: url } │                           │
 │                          ├─ Validate + map entity    │
 │                          ├─ INSERT product ─────────►│
 │◄── { product: {...} } ───┤                           │
```

### 6.2 Chi tiết xử lý trong `S3StorageServiceImpl.uploadImage()`

```
uploadImage(file, folder)
    │
    ├─ 1. validateFile()
    │       ├─ file != null && !isEmpty         → EMPTY_FILE
    │       ├─ contentType ∈ ALLOWED_TYPES      → INVALID_FILE_TYPE
    │       └─ size ≤ 2MB                       → FILE_TOO_LARGE
    │
    ├─ 2. buildS3Key(folder, filename)
    │       └─ "{folder}/{UUID}.{ext}"
    │          Ví dụ: "products/ed5b3da2-dd09-451c-bf78-eae28deb9e27.png"
    │
    ├─ 3. putObjectToS3()
    │       └─ PutObjectRequest
    │             .bucket(bucketName)
    │             .key(s3Key)
    │             .contentType(...)
    │             .acl(PUBLIC_READ)
    │
    ├─ 4. Build public URL
    │       └─ aws.base-url + "/" + s3Key
    │
    └─ 5. INSERT temp_uploads(url)
            └─ return UploadImageResponse { url }
```

## 7. Cleanup Job — Xóa ảnh mồ côi

`TempUploadCleanupService` chạy tự động lúc **2:00 AM mỗi ngày** để dọn các ảnh upload xong nhưng FE không dùng.

```
[2:00 AM Trigger]
    │
    ├─ Query: WHERE created_at < NOW() - 24h
    │
    ├─ Với mỗi orphan record:
    │       ├─ S3: DeleteObject(key)        # Xóa file vật lý
    │       ├─ DB: DELETE temp_uploads      # Xóa bản ghi
    │       └─ [Nếu lỗi]: log.error + tiếp tục record tiếp theo
    │
    └─ log: "Cleaned up X/Y orphan uploads"
```

> **Resilient:** Mỗi bản ghi xử lý độc lập — lỗi 1 bản ghi không làm dừng toàn bộ job.

### 8.3 Module User — Avatar

```json
// Bước 1: Upload
POST /api/v1/uploads/image/users
→ { "url": "https://.../users/uuid.png" }

// Bước 2: Cập nhật avatar vào profile
```

**Field lưu trong DB:** `users.avatar VARCHAR(500)`

**Xử lý đọc trong `UserServiceImpl.buildAvatarUrl()`:**
```java
// Nếu là S3 key: "users/uuid.png" → prepend base-url
// Nếu đã là full URL: "https://..." → giữ nguyên
private String buildAvatarUrl(String avatar) {
    if (avatar == null || avatar.isBlank()) return null;
    return avatar.startsWith("http") ? avatar : storageBaseUrl + "/" + avatar;
}
```

## 9. Thêm module mới tích hợp Upload

Để module mới (ví dụ: `Banner`) sử dụng Upload, chỉ cần **2 bước**:

**Bước 1 — Thêm hằng số folder** trong `UploadConstants.java`:
```java
/** Folder prefix cho ảnh banner trên S3. */
public static final String FOLDER_BANNERS = "banners";
```

**Bước 2 — Thêm endpoint** trong `UploadController.java`:
```java
@PostMapping(value = "/image/banners", consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
public ApiResponse<UploadImageResponse> uploadBannerImage(
        @RequestPart("file") MultipartFile file) {

    UploadImageResponse response = storageService.uploadImage(file, UploadConstants.FOLDER_BANNERS);

    return ApiResponse.<UploadImageResponse>builder()
            .message(Translator.toLocale("success.upload.image"))
            .result(response)
            .build();
}
```

> `S3StorageServiceImpl` và `TempUploadCleanupService` hoàn toàn **generic** — không cần chỉnh sửa.
